# クイックスタート

> **ひとことで：** ブランド名とドメインを登録するだけで、なりすましドメインの監視が始まります。

## 1. ログイン

管理者から招待されたアカウントで [ThreatGuard](https://threatguard.jp) にログインします。

::: info 招待制
ThreatGuard は招待制です。組織の管理者がユーザーを招待し、アカウントを発行します。公開登録（セルフサインアップ）には対応していません。
:::

- ログイン画面ではパスワードの **表示/非表示トグル** で入力内容を確認できます
- パスワードを忘れた場合は「パスワードを忘れた方」リンクから **パスワード再発行** が可能です

## 2. ブランド登録

初回ログイン後、保護したいブランドを登録します。

| 入力項目 | 説明 | 例 |
|---------|------|-----|
| ブランド名 | 保護したいサービス・ブランドの名称 | `サンプルブランド` |
| ロゴ画像 | ブランドのロゴ（アップロード可） | PNG / JPG |
| 商標登録証明 | 商標登録を証明する書類（アップロード可） | PDF / PNG / JPG |
| プライマリドメイン | フィッシング攻撃から守る対象ドメイン | `example-brand.com` |
| 保有ドメイン | ホワイトリスト登録＋期限切れ監視用 | `example-brand.co.jp,example-brand.net` |
| キーワード | 検知に使うキーワード（カンマ区切り） | `ExampleBrand,サンプルブランド,example-brand` |

::: tip ポイント
- キーワードには、ブランドの日本語名・英語名・略称・ティッカーなどを幅広く登録すると検知精度が上がります。
- ドメインは**プライマリ**（監視対象）と**保有**（ホワイトリスト＋期限切れ監視）の2層で管理されます。
- ロゴをアップロードすると、ブランド詳細ページで脅威統計サマリーとともに表示されます。
- 商標登録証明をアップロードすると、テイクダウン申請時の法的根拠が強化されます。
- 保有ドメインは**一括追加**に対応しています。複数ドメインをまとめて登録できます。
:::

## 3. セットアップチェックリスト

ブランド登録後、ブランド詳細ページに **セットアップチェックリスト** が表示されます。以下の4ステップを順番に完了していくことで、ThreatGuard を最大限に活用できます。

1. **ブランド情報の設定** — ロゴ・商標証明のアップロード
2. **ドメインの登録** — プライマリドメイン・保有ドメインの追加
3. **メール送信設定** — テイクダウンメールの送信元設定
4. **通知設定** — Slack Webhook・メール通知の設定

各ステップはページ内リンクでクリックするとそのセクションにジャンプします。

## 4. 自動スキャンの開始

ブランド登録が完了すると、ThreatGuard が **自動でフルスキャン** を開始します。プライマリドメインを追加した際にも自動的にスキャンが実行されます。

- **類似ドメイン生成 + DNS検証** — 登録・ドメイン追加時に自動実行
- **SSL証明書ログ監視** — 定期的に実行（設定可能）、CertSpotterフォールバック付き

::: tip ドメイン追加時の自動処理
保有ドメインを追加すると、該当ドメインが脅威として検出されていた場合、自動的に **false_positive（誤検知）** に再分類されます。手動で誤検知処理する必要はありません。
:::

## 5. 脅威の確認

ダッシュボードに検出された脅威が表示されます。

- 🔴 **対応が必要** — リスクスコアが高く、対応を推奨
- 🟡 **確認待ち** — 確認が必要な状態
- 🟢 **対応済み** — 削除申請完了 or 安全と判定

## 6. サイト調査

気になる脅威をクリックして詳細画面を開き、**STEP 2「🔍 サイトを調査する」** ボタンを押すと、ThreatGuard が自動でサイトにアクセスし：

- 画面キャプチャを取得
- ログインフォーム・パスワード入力欄の有無を検出
- ブランドキーワードの存在をチェック
- 公式サイトとの画像類似度を判定

## 7. 削除申請

脅威が確認できたら、**STEP 3「削除申請」** に進みます。

1. 対象ドメインを選択
2. abuse連絡先（送信先メールアドレス）を自動取得
3. AIが削除依頼文を自動生成（日本語/英語を自動判定）
4. 内容を確認して送信

送信後は「削除申請」画面で進捗を追跡できます。

::: tip ブラウザへの削除申請
レジストラへの削除申請と並行して、**Google Safe Browsing**（Chrome等）や **Microsoft SmartScreen**（Edge）にフィッシング報告を送信できます。ブラウザ側でブロックされることで、ドメイン停止前でもユーザーの被害を軽減できます。
:::
