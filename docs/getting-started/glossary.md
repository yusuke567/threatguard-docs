# 用語集

本ドキュメントで使用する用語の解説です。専門用語は初出時に説明を添えていますが、ここでまとめて確認できます。

## ブランド保護・フィッシング

| 用語 | 説明 |
|------|------|
| **フィッシング** | 偽のWebサイトやメールを使って、ユーザーのパスワードや個人情報を盗む詐欺手法 |
| **ブランドなりすまし** | 企業のロゴやデザインを模倣した偽サイトを作り、公式を装う行為 |
| **テイクダウン（削除申請）** | 不正なサイトやドメインの停止・削除を、ホスティング会社やレジストラに依頼すること |
| **ホモグリフ** | 見た目が似た別の文字。例: `o`（オー）と `0`（ゼロ）、`l`（エル）と `1`（イチ）。キリル文字の `а` とラテン文字の `a` など |
| **タイポスクワッティング** | ユーザーがURLを打ち間違えることを狙って、似たドメインを取得する手法。例: `gooogle.com` |

## ドメイン・DNS

| 用語 | 説明 |
|------|------|
| **ドメイン** | Webサイトのアドレス。例: `example-brand.com` |
| **DNS（Domain Name System）** | ドメイン名をIPアドレスに変換するインターネットの仕組み。電話帳のようなもの |
| **DNS解決（Resolve）** | ドメイン名からIPアドレスを取得すること。IPが返ってくる＝そのドメインにサーバーが存在する |
| **TLD（Top Level Domain）** | ドメインの末尾部分。`.com` `.net` `.org` `.jp` など |
| **レジストラ** | ドメインの登録を管理する事業者。例: お名前.com、GoDaddy |
| **WHOIS** | ドメインの登録者情報（所有者名、登録日、レジストラなど）を公開するプロトコル |
| **RDAP** | WHOISの後継プロトコル。HTTPベースでドメイン登録情報を照会する |

## SSL証明書

| 用語 | 説明 |
|------|------|
| **SSL証明書** | Webサイトの通信を暗号化し、身元を証明する電子証明書。`https://` のサイトが使用 |
| **Certificate Transparency（CT）** | SSL証明書の発行履歴を公開ログに記録する仕組み。誰がどのドメインの証明書を取得したかを追跡できる |
| **Let's Encrypt** | 無料でSSL証明書を発行するサービス。手軽なため、フィッシングサイトでも使用されやすい |
| **認証局（CA）** | SSL証明書を発行する機関。DigiCert、GlobalSign、Let's Encrypt など |

## セキュリティ

| 用語 | 説明 |
|------|------|
| **abuse連絡先** | 不正利用の通報を受け付けるメールアドレス。レジストラやホスティング事業者が設置 |
| **UDRP** | 統一ドメイン名紛争処理方針。ICANNが定めたドメイン名に関する紛争解決の国際ルール |
| **ICANN** | ドメイン名やIPアドレスの管理を行う国際機関 |
| **JPCERT/CC** | 日本のコンピュータセキュリティインシデントの調整機関。フィッシング情報の共有先として重要 |
| **pHash（Perceptual Hash）** | 画像の見た目の特徴をハッシュ値に変換する技術。2つの画像がどのくらい似ているかを数値で比較できる |

## ThreatGuard固有の用語

| 用語 | 説明 |
|------|------|
| **リスクスコア** | 0〜100の数値で脅威の危険度を表す。ドメイン類似度・ドメイン年齢・SSL情報・脅威カテゴリ・コンテンツ分析の5要素から算出 |
| **ページ調査（Probe）** | 不審ドメインに自動でアクセスし、画面キャプチャ・フォーム検知・コンテンツ分析を行う機能 |
| **脅威カテゴリ** | AIが判定する脅威の分類。phishing（フィッシング）/ brand_abuse（ブランド悪用）/ parked（パークドメイン）/ legitimate（正規）/ unknown（不明） |
| **ホワイトリスト** | 自社関連のドメインなど、脅威として検知しないように除外するリスト |
| **プライマリドメイン** | フィッシング攻撃から守る対象ドメイン。類似ドメイン検出・テイクダウン申請の対象となる |
| **保有ドメイン** | 自社が保有するドメイン。ホワイトリスト登録と期限切れ監視に活用される |
| **脅威確認（confirmed_threat）** | AI分析やページ調査により脅威と判定されたステータス。削除申請の対象となる |
| **商標登録証明** | ブランドの商標登録を証明する書類。アップロードすることでテイクダウン申請時の法的根拠を強化 |
| **セットアップチェックリスト** | ブランド詳細ページに表示される初期設定ガイド。ロゴ設定・ドメイン登録・メール設定・通知設定の4ステップで構成 |
| **監視ステータス** | ブランドごとのスキャン実行状態をリアルタイムで示す表示（スキャン中・待機中等） |
| **CertSpotter** | CT（Certificate Transparency）ログの検索サービス。ThreatGuardではメインCTログAPIのフォールバックとして使用 |
| **Google Safe Browsing** | Googleが提供するフィッシング・マルウェアサイトのブロックリスト。Chrome・Firefox・Safari等の主要ブラウザが参照し、危険なサイトにアクセスしようとするユーザーに警告を表示する |
| **SmartScreen（Microsoft SmartScreen）** | Microsoftが提供するフィッシング・マルウェアサイトのフィルタリング機能。Microsoft Edgeブラウザに組み込まれており、危険なサイトへのアクセスをブロックする |
