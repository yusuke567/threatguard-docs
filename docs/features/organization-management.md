# 組織・ユーザー管理

> **ひとことで：** マルチテナント対応の組織管理機能で、ブランド・脅威データを組織ごとに完全分離して管理します。

ThreatGuard はマルチテナントアーキテクチャを採用しており、組織（テナント）ごとにデータが厳密に分離されています。

## 組織管理

### 組織の構成

```
組織（Organization）
 ├── ユーザー（複数）
 ├── ブランド（複数）
 │    ├── 検出ドメイン
 │    ├── スキャンジョブ
 │    ├── フィッシングパターン
 │    └── SNS投稿
 └── 通知設定（Slack Webhook等）
```

### 組織レベルの設定

| 設定項目 | 説明 |
|---------|------|
| **組織名** | 組織の表示名 |
| **Slack Webhook URL** | Slack通知の送信先 |
| **Slack通知の有効/無効** | 通知のオン・オフ |
| **通知しきい値** | 通知するリスクスコアの下限 |
| **通知タイプ** | 通知する脅威カテゴリ |

## ユーザー管理

### ロール

| ロール | 権限 |
|--------|------|
| **member** | 所属組織のデータの閲覧・操作 |
| **superadmin** | 全組織のデータにアクセス可能、管理機能の利用 |

### データアクセスの分離

- `member` ロールのユーザーは、自分が所属する組織のブランド・脅威データのみにアクセスできます
- 他の組織のデータは一切参照できません
- `superadmin` は管理目的で全組織のデータにアクセスできます

### ユーザー設定

各ユーザーは以下の個人設定を管理できます：

| 設定項目 | 説明 |
|---------|------|
| **メール通知** | メールアラートの有効/無効 |
| **アラートしきい値** | 通知するリスクスコアの下限 |
| **パスワード変更** | パスワードリセット機能あり |

## 管理者画面

`/admin/organizations` 画面では、superadmin が以下の操作を行えます：

- 組織の一覧表示
- 組織の作成・編集
- ユーザーの組織への割り当て
- 組織ごとの通知設定

## アクティビティログ

システム管理者（superadmin）は、全ユーザーの操作履歴を監査ログとして閲覧できます。

| 項目 | 説明 |
|------|------|
| **操作内容（action）** | ログイン・ブランド作成・スキャン実行・テイクダウン送信等 |
| **操作者** | 操作を実行したユーザー名・メールアドレス |
| **操作日時** | タイムスタンプ |
| **対象** | 操作対象のリソース（entityType / entityId） |
| **メタデータ** | 操作に関する追加情報（JSON形式） |
| **IPアドレス** | 操作元のIPアドレス |
| **ユーザーエージェント** | 操作元のブラウザ/クライアント情報 |

## 認証

### ログイン

- メールアドレスとパスワードによるJWT認証
- トークン有効期限: 7日間
- 全APIリクエストでトークン検証を実施

### 自動ログアウト

セキュリティ強化のため、以下の条件で自動ログアウトが実行されます：

| 条件 | 説明 |
|------|------|
| **アイドルタイムアウト** | 一定時間操作がない場合に自動ログアウト |
| **セッション有効期限** | セッションの有効期限が切れた場合 |
| **警告ダイアログ** | タイムアウト前に警告を表示し、セッション延長の選択が可能 |

### パスワードリセット

1. ログイン画面から「パスワードを忘れた方」をクリック
2. メールアドレスを入力して送信
3. リセット用のリンクがメールで届く（有効期限: 1時間）
4. リンクから新しいパスワードを設定

::: info 技術詳細（CISO向け）
- 認証: JWT（jsonwebtoken）+ bcryptjs によるパスワードハッシュ化
- トークンペイロード: userId, email, name, organizationId, role
- パスワードリセットトークンはハッシュ化して保存
- レート制限: リセットトークンは5分間のクールダウン
- 全APIエンドポイントで `authMiddleware` による認証チェック
- 組織スコープ: `requireOrg` ミドルウェアでデータ分離を強制
- 管理者制限: `requireSuperAdmin` ミドルウェアで管理機能を保護
- 自動ログアウト: アイドルタイムアウト + セッション有効期限 + 警告ダイアログ
- アクティビティログ: `ActivityLog` モデルで全操作を記録、superadmin のみ閲覧可能
:::
