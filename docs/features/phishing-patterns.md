# フィッシングパターン報告

> **ひとことで：** ユーザーが発見したフィッシングサイトの情報を報告・共有し、組織全体の検知精度を向上させます。

ThreatGuard の自動検出に加えて、ユーザーが手動でフィッシングパターンを報告できる機能です。報告されたパターンは、リスクスコアリングの精度向上に活用されます。

## 報告の流れ

```
① パターン報告 → ② 情報の登録 → ③ リスクスコアへの反映 → ④ 組織内共有
```

## 報告できる情報

| 項目 | 説明 | 例 |
|------|------|-----|
| **パターン種別** | 攻撃の手法 | ドメインスプーフィング |
| **URL** | 不正サイトのURL | `https://examp1e-brand.com/login` |
| **ドメイン** | 不正サイトのドメイン | `examp1e-brand.com` |
| **説明** | 報告者による補足情報 | 「正規のログインページに酷似」 |
| **タグ** | 分類用のタグ | phishing, login-page |
| **被害件数** | 確認された被害の件数 | 5件 |

## グローバル検知ルール

複数の組織から報告された同一ドメインのパターンは、**グローバル検知ルール**として全組織で共有されます。

### グローバルルールの効果

グローバル検知ルールに登録されたドメインは、リスクスコアに**+20点のブースト**が加算されます。これにより、既知のフィッシングサイトの早期検出・高優先度対応が可能になります。

| ブースト要因 | 加算スコア |
|-------------|----------|
| グローバル検知ルール一致 | **+20点** |
| ユーザー報告あり | **+15点** |
| 被害件数（報告あり） | **+10点** |

## 管理画面

`/phishing-patterns` 画面で、ブランドごとのフィッシングパターンを一覧管理します。

- パターンの新規登録
- 既存パターンの編集・削除
- 重要度（severity）とステータスの管理
- タグによるフィルタリング

::: info 技術詳細（CISO向け）
- パターンはブランドに紐づいて管理（`PhishingPattern` モデル）
- グローバル検知ルールは組織を横断して共有（`GlobalDetectionRule` モデル）
- ドメイン単位でユニーク制約を適用
- リスクスコアリングエンジンがパターン情報を自動参照
:::
