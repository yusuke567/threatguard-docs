# レポート・エクスポート

> **ひとことで：** 検出した脅威や対応状況を、報告先に合わせた形式で自動レポート生成できます。ブラウザからPDF保存も可能です。

ThreatGuard では、提出先に応じて3種類のレポートを生成できます。報告書を手作業で作る必要はありません。

## レポートの種類

### 🏛️ 行政機関向け（Regulatory）

金融庁や当局への報告を想定した、全証拠付きの詳細レポートです。

**含まれる情報：**

| セクション | 内容 |
|-----------|------|
| サマリー | 検知脅威総数、危険レベル件数、削除申請済み・完了件数 |
| 検知タイムライン | 日別の脅威検知件数を棒グラフで表示 |
| 脅威一覧（全件） | ドメイン名、ブランド名、リスクスコア、脅威カテゴリ、ステータス、初検知日、削除申請状況 |

各脅威には以下の証拠データが紐付いています：
- WHOIS情報（登録者情報）
- スクリーンショット
- AI分析結果（カテゴリ・信頼度・判定理由）
- ページ調査結果（HTTPステータス・DNS解決・IPアドレス）
- 削除申請の送信日・回答日・レジストラ

### 📊 取締役会向け（Board）

KPIとビジュアルを中心にした、経営レベルの報告書です。

**含まれる情報：**

| セクション | 内容 |
|-----------|------|
| KPIカード | 検知脅威総数、危険レベル件数、削除申請成功率、高リスク件数 |
| リスク分布 | 危険・高・中・低の円グラフ |
| 脅威カテゴリ | フィッシング・ブランド悪用・パーク等の分布（円グラフ） |
| 検知トレンド | 日別の検知件数推移（棒グラフ） |
| ブランド別脅威件数 | 各ブランドの脅威件数をバーで比較 |

::: tip 活用シーン
月次の取締役会で「ブランド保護の状況」を報告する際に、このレポートをそのまま使えます。グラフ中心なので、短時間で状況を伝えられます。
:::

### ⚖️ CLO（法務責任者）向け

法的リスクの観点で整理した、法務部門向けのレポートです。

**含まれる情報：**

| セクション | 内容 |
|-----------|------|
| 高リスク脅威一覧 | リスクスコア60以上の脅威のみ抽出 |
| 法的根拠 | 各脅威に対する法的分類を自動付与 |
| 対応優先度 | 「最優先」（リスク80以上）/「要対応」（リスク60以上）で分類 |
| レジストラ別集計 | どのレジストラに何件の削除申請があるかを集計 |

**法的分類の自動付与：**

| 脅威カテゴリ | 法的根拠 |
|-------------|---------|
| phishing（フィッシング） | 不正競争防止法違反（詐欺的誘引） |
| brand_abuse（ブランド悪用） | 商標権侵害 |
| parked（パークドメイン） | ドメイン不正占拠（サイバースクワッティング） |
| unknown（不明） | 調査中 |
| legitimate（正規） | 侵害なし |

## レポートの生成方法

1. ナビゲーションの「📄 レポート」をクリック
2. ブランドを選択（全ブランド or 特定ブランド）
3. 3種類のレポートカードから、提出先に合わせたものをクリック
4. レポートが画面に表示される

## PDF保存

レポート表示後、画面右上の **「📥 PDF ダウンロード」** ボタンを押すと、ブラウザの印刷機能でPDF保存できます。印刷用のスタイルが適用され、操作ボタン等は非表示になります。

## ブランドフィルタ

レポートはブランドごとに生成できます。複数ブランドを管理している場合、特定ブランドに絞ったレポートを作成可能です。

::: info 技術詳細（CISO向け）
- レポート種別: regulatory（行政機関）/ board（取締役会）/ clo（法務責任者）
- マルチテナント対応: 組織内ブランドのみに制限
- システム管理者は全組織のデータを横断してレポート生成可能
- タイムラインは `firstSeen` の日付でグルーピング
- CLOレポートは `riskScore >= 60` の脅威のみ抽出
- グラフ描画: 棒グラフ・円グラフによるビジュアル表示
:::
