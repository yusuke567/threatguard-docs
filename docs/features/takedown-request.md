# 削除申請フロー

> **ひとことで：** フィッシングサイトのドメインを管理している事業者に「このサイトを停止してください」というメールを、AIが自動で作成して送信します。

脅威が確認できたら、次は削除申請です。ThreatGuard は、送信先の特定から文面作成、メール送信、進捗管理までをワンストップで提供します。

## 全体の流れ

```
① 対象選択 → ② 送信先の自動特定 → ③ AI文面生成 → ④ 確認・送信 → ⑤ 進捗追跡
```

## Step 1: 対象の選択

脅威一覧画面で、削除申請したいドメインをチェックボックスで選択します。**複数ドメインの一括申請**に対応しています。

## Step 2: abuse連絡先の自動取得

選択したドメインごとに、削除依頼の送信先（abuse連絡先）を自動で特定します。

### 特定の方法

2段階のフォールバック構成で、できるだけ連絡先を見つけます。

| 優先度 | 方法 | 説明 |
|--------|------|------|
| 1 | **WHOIS情報** | ドメインの登録情報に含まれるabuse連絡先メールを抽出 |
| 2 | **RDAP照会** | WHOISの後継プロトコルでHTTPベースの照会 |

::: info WHOIS/RDAPから抽出する情報
- **レジストラ名**（例: GoDaddy, お名前.com）
- **abuse連絡先メール**（例: `abuse@godaddy.com`）
- RDAPでは `vcardArray` から `registrar` ロールのエンティティ、`abuse` ロールのメールアドレスを構造的に取得
:::

### 日本のレジストラの自動判定

以下のレジストラは日本語での申請文を自動生成します：

- お名前.com（GMO）
- ムームードメイン（ペパボ）
- バリュードメイン
- エックスサーバー
- さくらインターネット
- JPRS
- ConoHa
- FC2
- その他（計18キーワードで判定：gmo, onamae, sakura, xserver, jprs, valuedomain, conoha, fc2, nic.ad.jp, gehirn, idcf, kagoya, cpi, wadax, interlink, nttpc, iij, biglobe）

### グルーピング

同じabuse連絡先に対して複数ドメインを申請する場合、自動的に1通のメールにまとめます。

```
例: GoDaddy の abuse@godaddy.com に3ドメインを一括申請
    → 1通のメールに3ドメイン分の申立をまとめて送信
```

## Step 3: AI文面生成

Claude（AI）が、ブランド情報と脅威情報をもとに、プロフェッショナルな削除依頼メールを自動生成します。

### 送信先の種類

| 送信先 | 説明 | 言語 |
|--------|------|------|
| **レジストラ** | ドメインを管理する事業者のabuse窓口 | 自動判定（日本語/英語） |
| **警察（警視庁サイバー犯罪対策課）** | 日本国内のサイバー犯罪通報 | 日本語 |
| **JPCERT/CC** | 日本のインシデント対応機関 | 日本語 |

### 文面に含まれる内容

1. **ブランド所有者情報** — 組織名、ブランド名、正規ドメイン
2. **不正ドメインの詳細** — ドメイン名、検出日、脅威カテゴリ、リスクスコア
3. **申立根拠** — 以下の国際ルール・ポリシーを引用
   - ICANN レジストラ認定契約（第3.18条）
   - 統一ドメイン名紛争処理方針（UDRP）
   - レジストラの利用規約（AUP）
   - APWG（Anti-Phishing Working Group）ベストプラクティス
4. **要請事項** — ドメインの即時停止、登録情報の保全、登録者情報の提供
5. **回答期限** — 72時間以内

### PDF出力

生成された削除申請テンプレートは **PDF形式でダウンロード** できます。PDFには以下が含まれます：

- 申請文面の全文
- エビデンス（スクリーンショット、商標登録証など）の添付
- ブランド情報と脅威情報のサマリー

社内承認フローや法務部門への共有に活用できます。

### 言語の自動切替

テンプレートの言語は、レジストラ名と国コード情報の2段階で自動判定します。

| 判定方法 | 条件 | 生成言語 | 宛先形式 |
|---------|------|---------|---------|
| レジストラ名 | 日本のレジストラ（お名前.com、ムームー等） | 日本語（敬語） | 「不正利用対応窓口 御中」 |
| 国コード | WHOIS/RDAPの国コードが `JP` | 日本語（敬語） | 「不正利用対応窓口 御中」 |
| その他 | 上記以外 | 英語 | 「Dear Abuse Department」 |

::: tip 国コードによる判定
v6より、レジストラ名だけでなくWHOIS/RDAPから取得した国コード情報でも言語を判定します。レジストラ名で判別できない場合でも、国コードが`JP`であれば日本語テンプレートが生成されます。
:::

### フォールバック

AIが使用できない場合（API障害等）は、事前に用意されたルールベースのテンプレートで文面を生成します。日本語・英語の両方に対応。

## Step 4: 確認・送信

生成された文面を画面上でプレビューし、必要に応じて編集した上でメール送信します。

- 送信はメールAPI / SMTP で abuse窓口に直接送信
- 送信結果はデータベースに記録
- 脅威のステータスが自動的に `takedown_sent`（削除申請済み）に更新

::: info SMTP エラー分類
メール送信失敗時は、エラーの種類に応じたメッセージが日本語で表示されます：
- **認証エラー**: メール送信設定のユーザー名・パスワードを確認してください
- **接続エラー**: メールサーバーへの接続に失敗しました
- **ブラウザエラー**: PDF生成用のブラウザ起動に問題があります
:::

## Step 5: 進捗追跡

`/takedowns`（削除申請）画面で、全申請の進捗を一元管理します。

### ステータス管理

| ステータス | 説明 |
|-----------|------|
| **draft** | 下書き（未送信） |
| **sent** | 送信済み |
| **acknowledged** | 受領確認（レジストラからの回答あり） |
| **completed** | 完了（ドメイン停止確認） |
| **rejected** | 却下（レジストラからの拒否） |

### 却下時の対応

レジストラから却下された場合：
1. 却下理由を記録
2. 文面を修正
3. **再送信**が可能（同じ画面から操作）

### 完了時の自動処理

削除申請が完了（ドメイン停止確認）すると、対応する脅威のステータスが自動的に `resolved`（解決済み）に更新されます。

## ブラウザへの削除申請

レジストラへの削除申請に加えて、主要ブラウザに対してフィッシングサイトのブロックを申請できます。ブラウザ側でブロックされることで、ドメインが停止される前でもユーザーが不審サイトにアクセスするのを防止できます。

### 対応ブラウザ

| ブラウザ | 報告先 | 効果 |
|---------|--------|------|
| **Google Chrome** | Google Safe Browsing | Chrome・Firefox・Safari 等、Safe Browsingを利用する全ブラウザでフィッシング警告が表示される |
| **Microsoft Edge** | Microsoft SmartScreen | Edge ブラウザでフィッシング警告が表示される |

### 仕組み

1. 脅威詳細画面またはテイクダウン画面から **ブラウザ削除申請** を選択
2. 脅威情報（URL・スクリーンショット・脅威カテゴリ等）を自動でフォーマット
3. Google Safe Browsing / Microsoft SmartScreen にフィッシング報告を送信
4. 申請ステータスをデータベースに記録・追跡

### レジストラ申請との併用

ブラウザへの削除申請は、レジストラへの削除申請と**併用**することを推奨します。

| 申請先 | 効果 | 対応速度 |
|--------|------|---------|
| レジストラ | ドメイン自体を停止 | 24〜72時間 |
| Google Safe Browsing | ブラウザでフィッシング警告を表示 | 数時間〜24時間 |
| Microsoft SmartScreen | Edgeでフィッシング警告を表示 | 数時間〜24時間 |

ブラウザのブロックはレジストラ対応より早く反映されることが多く、ドメイン停止までの被害を軽減する効果があります。

::: tip 推奨フロー
1. **即時対応** — Google Safe Browsing + SmartScreen にブラウザ削除申請
2. **並行して** — レジストラにドメイン停止の削除申請
3. **追跡** — 両方の進捗をダッシュボードで一元管理
:::

## サマリー統計

削除申請画面には、全体のサマリーが表示されます：
- 総申請数
- 下書き / 送信済み / 回答待ち / 完了 / 却下 の件数
- ブラウザ申請数（Google Safe Browsing / SmartScreen）

::: info 技術詳細（CISO向け）
- メール送信: メールAPI（Resend）+ SMTP（ブランド別設定対応）
- バッチ処理: 複数申請をグルーピングして一括管理
- WHOIS解析: JSON形式とプレーンテキスト形式の両方をパース
- RDAP: 公開RDAPブートストラップ経由でのドメイン情報照会
- テンプレート生成: LLM API による自動生成
- 再送信時: ステータスをリセットし、却下理由をクリアして再申請
- ブラウザ削除申請: Google Safe Browsing Report API + Microsoft SmartScreen 報告
:::
