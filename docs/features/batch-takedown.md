# 一括テイクダウン

> **ひとことで：** 複数のフィッシングサイトに対する削除申請をまとめて作成・送信・管理できるバッチ処理機能です。

大量の脅威が検出された場合、1件ずつ削除申請を行うのは非効率です。一括テイクダウン機能を使えば、複数ドメインの削除申請をバッチとしてまとめて管理できます。

## バッチの流れ

```
① 対象ドメインを複数選択 → ② バッチ作成 → ③ 一括送信 → ④ 進捗の一元管理
```

## バッチのステータス

| ステータス | 説明 |
|-----------|------|
| **draft** | 下書き（送信前） |
| **sent** | 全件送信済み |
| **partial** | 一部送信済み（残りは未送信または失敗） |
| **completed** | 全件完了（ドメイン停止確認） |

## バッチの管理情報

各バッチには以下の集計情報が表示されます：

- **総件数**（totalCount）— バッチに含まれる申請の総数
- **送信済み件数**（sentCount）— 実際に送信された申請の件数
- 個別の申請ステータス — バッチ内の各申請の詳細状態

## 自動グルーピング

同じレジストラ（abuse連絡先）に対する複数ドメインの申請は、自動的に1通のメールにグルーピングされます。これにより、レジストラ側の対応効率も向上します。

```
例: バッチに10ドメインを選択
    → GoDaddy 管理: 4ドメイン → 1通にまとめて送信
    → お名前.com 管理: 3ドメイン → 1通にまとめて送信（日本語）
    → Namecheap 管理: 3ドメイン → 1通にまとめて送信（英語）
```

## 送信順序

バッチテイクダウンでは、以下の推奨順序でUIに表示・送信されます：

| 順序 | 送信先 | 説明 |
|------|--------|------|
| 1 | **ブラウザ** | Google Safe Browsing / Microsoft SmartScreen（即時ブロック効果） |
| 2 | **レジストラ** | ドメイン停止のための削除申請 |
| 3 | **警察** | 警視庁サイバー犯罪対策課への通報 |
| 4 | **JPCERT/CC** | インシデント対応機関への通報 |

## CSVからの一括インポート

CSVファイルからドメインを一括インポートし、DetectedDomain（検出ドメイン）に自動適用できます。大量のドメインを効率的にバッチに追加する場合に便利です。

## 使い方

1. 脅威一覧画面で対象ドメインをチェックボックスで複数選択（またはCSVから一括インポート）
2. 「一括テイクダウン」ボタンをクリック
3. バッチが作成され、各ドメインの削除申請テンプレートが**並列で**自動生成
4. 内容を確認後、一括送信
5. バッチ画面で進捗を追跡

::: info 技術詳細（CISO向け）
- バッチは `TakedownBatch` モデルで管理
- 個別の申請は `TakedownRequest` モデルの `batchId` でバッチに紐づけ
- バッチステータスは個別申請のステータスから自動算出
- レジストラごとのグルーピングは abuse連絡先メールアドレスで判定
- テンプレート生成は並列実行（タイムアウト対策付き）
- abuse連絡先が見つからないレジストラもUI上で可視化（手動対応を促進）
- CSV一括インポートは `DetectedDomain` に自動適用
:::
