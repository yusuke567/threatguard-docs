# SNS監視

> **ひとことで：** Twitter/X などのSNSで、ブランド名を騙った不正アカウントや偽サイトへの誘導投稿を自動検出します。

ドメインスキャンがWebサイトを対象とするのに対し、SNS監視はソーシャルメディア上のブランドなりすましを検出する機能です。

## 対応プラットフォーム

| プラットフォーム | 監視内容 |
|----------------|---------|
| **Twitter / X** | ブランド名を含む投稿、不審なURLの共有 |
| **Facebook** | なりすましページ、偽広告 |
| **Instagram** | 偽アカウント、フィッシングリンク |
| **Telegram** | 偽チャンネル、詐欺メッセージ |
| **Discord** | 偽サーバー、フィッシングリンク |

## 検出の仕組み

### 1. キーワードモニタリング

ブランドに登録されたキーワード（ブランド名、ドメイン名など）をもとに、SNS上の投稿を定期的に検索します。

### 2. URL抽出と照合

投稿内に含まれるURLを抽出し、以下の条件で不審な投稿を判定します：

- 投稿内のURLが、検出済みの不審ドメインと一致する
- ブランドのドメインに類似したURLが含まれている
- 短縮URLの展開先が不審なドメインを指している

### 3. リスク評価

検出された投稿ごとにリスクスコアを算出し、以下のステータスで管理します：

| ステータス | 説明 |
|-----------|------|
| **new** | 新規検出 |
| **reviewed** | 確認済み（対応不要または対応完了） |
| **dismissed** | 誤検知として除外 |

## 投稿の管理画面

`/social-monitor` 画面で、検出されたSNS投稿を一覧管理できます。

各投稿には以下の情報が表示されます：

- **プラットフォーム** — 投稿元のSNS
- **投稿者** — アカウント名・ハンドル
- **投稿内容** — テキスト本文
- **含まれるURL** — 投稿内のリンク
- **一致ドメイン** — ThreatGuardで検出済みのドメインとの一致
- **リスクスコア** — 0〜100
- **投稿日時** — SNS上での投稿時刻

## 通知連携

高リスクの投稿が検出された場合、Slack通知やメール通知と連携して即座にアラートを送信します。

::: info 技術詳細（CISO向け）
- Twitter/X API を利用したリアルタイム検索
- 定期スケジューラによるバッチ監視（スキャン間隔はcron設定で調整可能）
- 投稿内URLは正規化した上でドメイン照合を実施
- 組織・ブランド単位でのフィルタリング
:::
