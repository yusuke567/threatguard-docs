# ページ調査（Web Probe）

> **ひとことで：** 不審なドメインに自動でアクセスし、画面キャプチャを撮影して、ログインフォームや自社キーワードの有無を調べます。

ドメインスキャンで「ドメインが存在する」ことはわかりましたが、実際にどんなサイトが表示されるかはまだわかりません。ページ調査では、ヘッドレスブラウザ（画面を表示しないブラウザ）を使って自動的にサイトにアクセスし、内容を調査します。

## 調査の流れ

脅威詳細画面の **STEP 2「🔍 サイトを調査する」** ボタンを押すと、以下の調査が自動実行されます。

### 1. DNS解決

ドメインのIPアドレスを取得します。

- IPアドレスが返る → サーバーが存在する
- 返らない → ドメインは登録されているがサーバー未設定

### 2. サイト訪問

ヘッドレスブラウザでサイトにアクセスします。

- まず `https://` でアクセスを試行
- 失敗した場合は `http://` にフォールバック
- ページの読み込み完了（ネットワーク通信が落ち着くまで）を最大30秒待機
- リダイレクトがあれば最終URLを追跡・記録

### 3. 画面キャプチャ

1280×720ピクセルのビューポートでスクリーンショットをPNG形式で取得します。タイムアウトした場合も、読み込み途中の画面をキャプチャします。

### 4. 情報収集

| 収集項目 | 内容 |
|---------|------|
| HTTPステータスコード | `200`（正常）、`404`（未発見）、`403`（アクセス拒否）など |
| 最終URL | リダイレクト先のURL（元のドメインと異なる場合がある） |
| IPアドレス | サーバーのIPアドレス |
| レスポンスヘッダー | サーバーの種類や設定情報（全量取得） |
| HTMLスニペット | ページのHTMLソースコードの先頭5,000文字 |

## コンテンツ分析

取得したHTMLを解析して、フィッシングの兆候を4つの観点で検出します。

### ログインフォーム検知

HTMLの `<form>` タグ内に以下のキーワードが含まれるかをチェック：
- `login`, `signin`, `sign-in`, `log-in`
- `type="password"`

::: warning なぜ重要か
フィッシングサイトの多くは、ユーザーにログイン情報を入力させて盗むことが目的です。偽サイトにログインフォームがある場合、フィッシングの可能性が高まります。
:::

### パスワードフィールド検知

HTML内に `type="password"` の入力フィールドがあるかをチェックします。ログインフォームと組み合わせて判定します。

### ブランドキーワードマッチ

ブランド登録時に設定したキーワード（例: `サンプルブランド`, `ExampleBrand`, `example-brand`）がHTML内に含まれるかを照合します。

- 偽サイトのHTMLに自社ブランド名が含まれている → なりすましの可能性が高い

### ロゴ画像検知

`<img>` タグや `background-image` CSS で `logo` を含む画像要素があるかをチェックします。

## 調査結果の見方

調査完了後、脅威詳細画面に以下が表示されます。

### 概要エリア（STEP 2）

サイトの状態を3段階で表示：
- **サイトが稼働中** — HTTPステータス200が返った
- **ドメインは存在するがサイトは応答しない** — DNS解決はできたがHTTPアクセス失敗
- **サーバー未設定** — DNS解決ができない

### 詳細エリア（「📋 調査の詳細を見る」を展開）

- ✅ / ⚠️ でフィッシング兆候の有無をチェックリスト表示
  - ログイン画面の模倣
  - パスワード入力欄
  - 自社ロゴの無断使用
  - 自社サイトとの類似度（%表示）
  - 不審なキーワード（検出されたキーワードをタグ表示）
- 画面キャプチャ（撮影日時付き）

### スクリーンショットタブ

画面キャプチャの大きな表示と、再調査ボタン。

### 技術情報タブ（CISO向け）

| 項目 | 内容 |
|------|------|
| DNS解決 | 解決済み / 未解決 |
| IPアドレス | サーバーIP |
| HTTP Status | ステータスコード |
| 最終URL | リダイレクト先 |
| レスポンスヘッダー | JSON形式で全量表示 |
| HTMLスニペット | ソースコードの先頭5,000文字 |

::: info 技術詳細（CISO向け）
- ブラウザエンジン: Chromium ベースのヘッドレスブラウザ
- ビューポート: 1280×720
- HTTPS → HTTP のフォールバック対応
- スクリーンショット: PNG形式（ビューポート内）
- 調査結果は履歴として保存（上書きではなく追記）
:::

## 再調査

サイトの内容は時間とともに変化します。「🔄 再調査する」ボタンで最新の状態を再取得できます。過去の調査結果は履歴として保持されます。
