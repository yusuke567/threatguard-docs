# 画像類似度判定

> **ひとことで：** 不審サイトの画面キャプチャと自社公式サイトの画面を、画像の「見た目」で比較して、どのくらい似ているかを数値化します。

フィッシングサイトは、公式サイトのデザインをそっくりコピーすることが一般的です。画像類似度判定では、HTMLの文字列比較ではなく、**画面の見た目そのもの**を比較することで、巧妙に作られた偽サイトも検出します。

## 仕組み

### 画像比較アルゴリズム

2つのスクリーンショットをリサイズ・グレースケール化した上で、ピクセル単位の差異を算出して類似度を求めます。

```
不審サイトのスクリーンショット → 16×16 グレースケール化 → 画像データA
                                                              ↓ 比較（MAE）
公式サイトのスクリーンショット  → 16×16 グレースケール化 → 画像データB
                                                              ↓
                                                        類似度: 0〜1
```

- **1.0** = 完全に同じ画像
- **0.7以上** = かなり似ている（⚠️ 警告表示）
- **0.3以下** = ほとんど別のサイト

### アルゴリズムの特徴

| 特徴 | 説明 |
|------|------|
| サイズ不問 | 16×16にリサイズして正規化するため、元の解像度に依存しない |
| 色合いで比較 | グレースケール変換後のピクセル差分（MAE: Mean Absolute Error）で比較 |
| テキスト変更に強い | 全体のレイアウトや明暗が同じなら、文字の違いは影響しにくい |
| 高速 | 16×16 = 256ピクセルの比較のみ（ミリ秒単位） |

### リファレンス画像（参照画像）

公式サイトのスクリーンショットは、ブランド登録後に初回自動取得され、`screenshots/reference/` にキャッシュされます。

- 初回: 公式ドメインにヘッドレスブラウザでアクセスし、スクリーンショットを自動取得
- 以降: キャッシュ済みの画像と比較

## 判定結果の見方

脅威詳細画面の「調査の詳細」内に表示されます。

- **⚠️ 自社サイトとの類似度: 85%** → 高い類似度。なりすましの可能性が高い
- **✅ 自社サイトとの類似度: 15%** → 類似度が低い。別のサイトの可能性

## スコアへの影響

画像類似度は [リスクスコアリング](/features/risk-scoring) の「コンテンツ分析」要素に含まれ、最大25ポイントの加点に寄与します。

::: info 技術詳細（CISO向け）
- 画像処理ライブラリ: sharp（libvips ベース）
- 前処理: 16×16ピクセルにリサイズ → グレースケール変換
- 比較方法: MAE（Mean Absolute Error / 平均絶対誤差）
- 類似度計算: `1 - (MAE / 255)`（0〜1の範囲に正規化）
- リファレンス画像は初回アクセス時にキャッシュ
- キャプチャ不可（タイムアウト等）の場合はスコア計算から除外
:::
