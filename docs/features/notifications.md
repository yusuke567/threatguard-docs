# 通知設定

> **ひとことで：** 脅威の検出やサイトの変化を、Slackやメールでリアルタイムにチームへ通知します。

ThreatGuard は、重要なイベントが発生した際に複数のチャネルで通知を送信します。通知の条件やしきい値は、組織・ユーザー単位でカスタマイズできます。

## 通知チャネル

### Slack通知

組織ごとにSlack Webhook URLを設定し、指定チャンネルにリアルタイム通知を送信します。

#### 通知される内容

| イベント | 通知内容 |
|---------|---------|
| **新規脅威検出** | ドメイン名、リスクスコア、脅威カテゴリ |
| **サイト変化検出** | DNS変更、HTTPステータス変化、フォーム検出 |
| **スキャン完了** | スキャン結果のサマリー（検出数、高リスク数） |

#### リスクレベル表示

通知にはリスクレベルに応じた絵文字が付与されます：

| リスクスコア | レベル | 表示 |
|------------|--------|------|
| 80〜100 | 危険 | :red_circle: |
| 60〜79 | 高 | :orange_circle: |
| 40〜59 | 中 | :yellow_circle: |
| 0〜39 | 低 | :green_circle: |

#### しきい値設定

組織ごとに通知のしきい値を設定できます。例えば「リスクスコア60以上の脅威のみ通知」のように、重要度に応じたフィルタリングが可能です。

#### 通知タイプの選択

通知する脅威カテゴリを選択できます：
- フィッシング（phishing）
- ブランド悪用（brand_abuse）
- パークドメイン（parked）
- すべて

### メール通知

ユーザーごとにメール通知の有効/無効としきい値を設定できます。

#### 通知テンプレート

メール通知はHTMLテンプレートで送信され、以下の情報が含まれます：

- 脅威の概要（ドメイン、カテゴリ、リスクスコア）
- リスクレベルに応じた色分け表示
- ダッシュボードへのリンク
- 対象ブランド情報

#### ブランド別SMTP

ブランドごとに独自のSMTPサーバーを設定できます。これにより、ブランドの公式メールアドレスから通知を送信することが可能です。

| 設定項目 | 説明 |
|---------|------|
| 送信元メール | ブランドのメールアドレス |
| SMTPホスト | メールサーバーのアドレス |
| SMTPポート | ポート番号（通常465または587） |
| SMTPユーザー | 認証ユーザー名 |
| SMTPパスワード | 認証パスワード |

ブランド別SMTPが未設定の場合は、共有のメールサービス（Resend）経由で送信されます。

## 設定方法

### 組織のSlack通知設定

管理者が組織設定画面で以下を設定します：

1. **Slack Webhook URL** — Slackの「Incoming Webhooks」で取得したURL
2. **通知の有効/無効** — Slack通知のオン・オフ
3. **しきい値** — 通知するリスクスコアの下限値
4. **通知タイプ** — 通知する脅威カテゴリの選択

### ユーザーのメール通知設定

各ユーザーが `/alerts` 画面で以下を設定します：

1. **メール通知の有効/無効**
2. **しきい値** — 通知するリスクスコアの下限値

::: info 技術詳細（CISO向け）
- Slack通知: Webhook URLへのHTTP POST（JSON payload）
- メール通知: Resend API またはブランド別SMTP（nodemailer）
- 通知は定期スキャンの完了時に自動トリガー
- 組織単位 + ユーザー単位の二重フィルタリング
- アラートログ（`AlertLog` モデル）で送信履歴を管理
:::
