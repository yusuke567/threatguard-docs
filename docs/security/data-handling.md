# データの取り扱い

> **ひとことで：** ThreatGuard が収集・保存するデータの種類と、その管理方法について説明します。

## 収集するデータ

### ブランド情報（ユーザーが登録）

| データ | 説明 |
|--------|------|
| ブランド名 | 保護対象のブランド名称 |
| 正規ドメイン | 公式サイトのドメイン |
| キーワード | 検知に使用するキーワード |
| ホワイトリスト | 検知対象外のドメイン |
| 公式サイトスクリーンショット | 画像類似度判定の参照画像 |

### 脅威情報（自動収集）

| データ | 収集元 | 説明 |
|--------|--------|------|
| 検出ドメイン名 | ドメインスキャン / CT監視 | 不審なドメイン名 |
| DNS情報 | DNS解決 | IPアドレス、解決の成否 |
| WHOIS情報 | WHOIS / RDAP | 登録者情報、レジストラ、abuse連絡先 |
| SSL証明書情報 | CT監視 | 発行元、有効期間 |
| HTTPレスポンス | ページ調査 | ステータスコード、ヘッダー |
| HTMLスニペット | ページ調査 | ページソースの先頭5,000文字 |
| スクリーンショット | ページ調査 | 1280×720 PNGキャプチャ |
| AI分析結果 | Anthropic API | カテゴリ、信頼度、判定理由 |

### ユーザー情報

| データ | 説明 |
|--------|------|
| メールアドレス | ログイン認証 |
| パスワード | ハッシュ化して保存 |
| 組織情報 | テナント分離 |
| ロール | アクセス制御 |

## データの保存

### 保存場所

| データ種別 | 保存先 | 暗号化 |
|-----------|--------|--------|
| 構造化データ（脅威情報・ユーザー情報等） | マネージドRDB | 転送中: TLS |
| セッション・キャッシュ | マネージドキャッシュ | 転送中: TLS |
| スクリーンショット | 永続ボリューム（`DATA_DIR`） | — |
| 商標登録証明 | 永続ボリューム（`DATA_DIR`） | — |
| 静的アセット | CDN | 転送中: TLS |

### データの分離

- **テナント分離**: 各組織のデータは `organizationId` で分離され、他の組織からアクセスできません
- **APIレベルのフィルタリング**: 全APIエンドポイントでテナントチェックを実施

## データの保持と削除

| データ | 保持期間 |
|--------|---------|
| 脅威検出データ | 無期限（履歴として保持） |
| ページ調査結果 | 無期限（履歴として保持） |
| 削除申請データ | 無期限（監査証跡として保持） |
| スクリーンショット | 無期限 |
| 認証トークン | セッション有効期間内 |

## 外部サービスへのデータ送信

| 送信先 | 送信データ | 目的 |
|--------|-----------|------|
| AI API | ドメイン情報、WHOIS情報、ブランド情報 | AI脅威分類、文面生成 |
| CTログサービス | ブランド名・キーワード（検索クエリ） | CT監視 |
| RDAP | ドメイン名 | 登録情報照会 |
| メールAPI / SMTPサーバー | テイクダウンメール文面・通知メール | 削除依頼の送信・アラート通知 |
| Slack Webhook | アラートメッセージ | 組織ごとのSlack通知 |

::: warning 注意事項
AI分析（Anthropic API）に送信するデータには、検出ドメインの情報（ドメイン名、WHOIS、SSL）とブランド基本情報が含まれます。ユーザーの個人情報やログイン情報は送信しません。
:::
