# 変更履歴

ThreatGuard ドキュメントおよびプロダクトの変更履歴です。

## 2026-04-21 — ドキュメント v8 アップデート

警視庁サイバー犯罪対策課の警部への取材により、同課への通報ではフィッシングサイトの削除（テイクダウン）は実施されないことが確認されました。この事実を踏まえ、誤解を招かないよう、削除申請・一括削除申請の送信先から警視庁サイバー犯罪対策課を削除しました。

### 🗑️ 機能削除

| 項目 | 説明 |
|------|------|
| **警視庁サイバー犯罪対策課への削除申請** | 個別削除申請・一括削除申請・Step1/2/3のすべてから、警視庁サイバー犯罪対策課宛ての送信機能を削除。警視庁サイバー犯罪対策課は通報先であり、フィッシングサイトの削除（テイクダウン）業務は実施していないため |

### 🔧 改善

| 項目 | 説明 |
|------|------|
| **削除申請送信先の順序を更新** | 一括・個別ともに `ブラウザ → レジストラ → ホスティング → JPCERT/CC` の順序に変更（警視庁を除外） |

### 📝 ドキュメント変更

| ページ | 変更内容 |
|--------|---------|
| 削除申請フロー | 送信先テーブルから「警察（警視庁サイバー犯罪対策課）」行を削除。送信先の標準順序を更新 |
| 一括テイクダウン | 送信順序テーブルから「警察」行を削除し、JPCERT/CCを4番目に繰り上げ |
| MVP機能一覧 | 送信先の統一順序の記載から警視庁を削除 |
| 料金プラン / プラン一覧 | 「警察・JPCERT/CC通報」を「JPCERT/CC通報」に更新 |
| FAQ | 送信先順序に関するQ&Aから警視庁の記載を削除 |
| 変更履歴 | 本エントリを追加 |

::: info 背景
警視庁サイバー犯罪対策課は、サイバー犯罪被害の**通報・相談窓口**として機能しますが、フィッシングサイトそのものを削除する権限・業務は持ちません。従来のドキュメントでは「削除申請」の送信先に含めていたため、実態と一致するよう修正しました。なお、被害相談・通報先としての外部窓口情報は [セキュリティ / コンプライアンス](/security/compliance) ページに引き続き記載しています。
:::

## 2026-04-14 — ドキュメント v7 アップデート

4月10日〜14日のプロダクト変更を反映し、関連ページを最新化しました。ホスティング事業者への削除申請、JPCERT/CCフィード連携、攻撃インテリジェンス（Layer 3 / Layer 4）など、大型機能追加を含むアップデートです。

### 🆕 新機能

| 機能 | 説明 |
|------|------|
| **ホスティング事業者への削除申請** | IP-based RDAP でホスティング事業者の abuse 連絡先を特定し、AUP違反を根拠としたコンテンツ削除要請テンプレートを自動生成。個別・一括の両方に対応 |
| **JPCERT/CCフィッシングURLフィード連携** | JPCERT/CC 公開の phishurl-list を日次03:00 JST で自動取り込み（`KnownPhishingUrl`）。リスクスコアへ +30 点のブースト、無料診断の即時判定に利用 |
| **JPCERT由来アラート（Pro+限定）** | 取り込んだ JPCERT 観測URLのうち、登録ブランドに該当するものを自動で脅威登録し通常アラートを発報。Starter ではロック表示 |
| **Layer 3 攻撃インテリジェンスパネル（Pro+限定）** | ブランド詳細にあなたのブランドへの過去の攻撃傾向を可視化：月次タイムライン（過去24ヶ月）/ TLD分布 / URLパス傾向 / 偽装ブランド名バリアント / ピーク月 |
| **Layer 4 JPCERTコーパス検知パターン学習** | JPCERT 観測URLから n-gram / TLD / パスプレフィックス / サブドメインパターンを自動抽出し、未観測ドメインのリスク判定に活用（+最大30点） |
| **フィード取り込み手動実行API** | `POST /api/admin/feed-imports/run` により JPCERT 取り込みを即時実行可能（二重起動防止・バックグラウンド実行） |
| **組織プラン管理** | `Organization.plan`（starter/professional/enterprise/enterprise_plus）フィールド追加。Pro+限定機能の可視化・アップセルUIに使用 |

### 🔧 改善

| 項目 | 説明 |
|------|------|
| **削除申請送信先の順序統一** | 一括・個別ともに `ブラウザ → レジストラ → ホスティング → 警視庁 → JPCERT` の順に統一 |
| **一括削除申請UIの整理** | レジストラ・ホスティング宛先リストをデフォルトで折りたたみ表示。社数バッジは常時表示 |
| **backfill-whois の retry モード** | `?retry=true` で `rdap_failed` レコードのみを対象に効率的にリトライ（`refresh=true` の全件再取得との使い分け） |
| **スケジューラ拡張** | 日次 JPCERT 取り込みジョブと、取り込み後のパターン再学習を自動連結（非致命エラーで継続） |

### 📝 ドキュメント変更

| ページ | 変更内容 |
|--------|---------|
